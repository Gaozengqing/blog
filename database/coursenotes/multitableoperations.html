# Cross-join query

Table A has three fields and four records, and Table B has two fields and three records The resulting table has 3+2 fields and 4*3 records

Complete writing:
SELECT *|{field name [,...] } FROM Table 1 CROSS JOIN Table 2;
Abbreviation:
SELECT *|{field name [,...] } FROM Table 1, Table 2;

# Inner join query

Saves the intersection of two tables

SELECT *|{field name [,...] } FROM Table 1 [INNER] JOIN Table 2 ON join conditions;

# Self-query

A data table connected to itself is physically the same data table, but logically divided into two data tables. A self-join query must set an alias for the table, which is used to distinguish two data tables.

SELECT *|{field name [,...] } FROM Table 1 Aliases 1 JOIN Table 1 aliases 2 ON Connection conditions.

# Joint query

ALL Keyword: All query results are reserved. DISTINCT keyword: The default option is used to remove duplicate data in the query results. The fields of the federated query are derived from the first query statement, and the number of fields in multiple federated queries must be the same (the number of fields in the first table must be the same).

SELECT *|{field name [,...] } FROM Table 1...
UNION [ALL|DISTINCT]
SELECT *|{field name [,...] } FROM Table 2... ;

# external connection

# outside left

All of the left table plus the intersection of the left and right tables The records in the left table are matched with the records in the right table. The query result includes all the records in the left table and the records that meet the connection conditions in the right table. If a record in the left table does not exist in the right table, the value of the corresponding field in the right table appears as NULL.

SELECT table name. Field name [,...] FROM the LEFT table LEFT [OUTER] JOIN the right table ON the join condition;
1

# outside right

All of the right table plus the intersection of the left and right tables The records in the right table are matched with the records in the left table. The query result includes all the records in the right table and the records that meet the connection conditions in the left table. If a record in the right table does not exist in the left table, the value of the corresponding field in the left table appears as NULL.

SELECT table name. Field name [,...] FROM the left table RIGHT [OUTER] JOIN the right table ON the join condition;
1

# Tip

Left-outside join queries and right-outside join queries are interchangeable. When LEFT JOIN and RIGHT JOIN replace each other, the positions of the left and right tables are replaced with each other to get the same result.

# Subquery

# quantum query

A subquery that returns rows and columns of data.

SELECT *|{field name [,...] } FROM table name WHERE field name {=|<>|>|>=|<|<=}
(SELECT field name FROM table name [WHERE] [GROUP BY] [HAVING] [ORDER BY] [LIMIT]);

# column subquery

A subquery that returns multiple rows of data in one column. Common operators for column subqueries:

  1. in Within the specified range of collections, choose more than one
  2. not in Not within the specified range of collections
  3. any In the subquery return list, any of the conditions can be met
  4. some Same as any or so
  5. all All values returned by the subquery must be satisfied
SELECT *| field FROM table name WHERE field name {IN|NOT IN|ANY|SOME|ALL}
(SELECT field name FROM table name [WHERE] [GROUP BY] [HAVING] [ORDER BY] [LIMIT]);
1
2

# line subquery

Returns a subquery with multiple columns of data in one row.

SELECT *|{field name [,...] } FROM table name WHERE {Specify field name 1, specify field name 2,... } {=|<>|IN|NOT IN} (SELECT field name [,... FROM table name [WHERE] [GROUP BY] [HAVING] [ORDER BY] [LIMIT]);
1

# Table subquery

A subquery that returns multiple rows and columns of data. Table subqueries are mostly located after the FROM keyword, and the queried data is regarded as the data source.

SELECT *| field name [,...] FROM (table subquery) [AS] alias [WHERE] [GROUP BY] [HAVING] [ORDER BY] [LIMIT];
1

# EXISTS subquery

Used to determine whether a record that meets a given condition exists.

SELECT *| field name [,...] FROM table name WHERE EXISTS (SELECT * FROM table name [WHERE] [GROUP BY] [HAVING] [ORDER BY] [LIMIT]);
